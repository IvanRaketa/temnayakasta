# Production: Vercel + Neon

## Текущая схема

- веб-приложение размещается на Vercel;
- рабочая PostgreSQL-база размещается в Neon;
- основной публичный и canonical-адрес: `https://temnayakasta120.online`;
- дополнительный домен: `temnayakasta120.ru`; он может перенаправлять на основной адрес после
  настройки DNS;
- Cloudflare, Cloudflare Tunnel, домашний компьютер и локальный PostgreSQL не входят в текущий
  production-контур.

Фиксированный IP не указывается: адреса платформенной инфраструктуры могут меняться. Для публичной
и правовой документации используются названия провайдеров и домены.

## Переменные окружения

Production-значения хранятся только в Vercel environment variables. В репозиторий не должны
попадать реальные `DATABASE_URL`, `DIRECT_URL`, SMTP-пароли, ключи, токены или иные секреты.

Безопасные имена и примеры перечислены в `.env.example`. Строка `DATABASE_URL` предназначена для
прикладного подключения к Neon, а `DIRECT_URL` — только для процессов, которым явно требуется
прямое подключение. Конкретные значения берутся из настроек Neon и не публикуются.

## Обработка данных

Vercel и Neon являются инфраструктурными провайдерами. При их использовании возможна техническая
обработка данных с применением иностранной инфраструктуры и передача за пределы Российской
Федерации. Конкретные страны и юрисдикции зависят от настроек проекта, договоров и инфраструктуры
провайдеров.
